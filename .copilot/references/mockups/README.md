# Mockups

Place UI mockups and design assets here so Copilot and teammates can reference the intended look and feel during development.

## What belongs here

- **Wireframes** — low-fidelity layout sketches for screens or components
- **High-fidelity mockups** — polished designs exported from Figma, Sketch, or similar
- **Annotated screenshots** — marked-up images explaining behaviour or edge cases
- **Component specs** — spacing, colour, and typography breakdowns for complex UI elements
- **User flow diagrams** — how users navigate between screens

## Recommended formats

- `.png` / `.jpg` — rendered exports for quick viewing
- `.svg` — scalable vector exports from design tools
- `.fig` — Figma source files (if the team uses Figma)
- `.pdf` — multi-page design handoff documents

## Tips

- Name files by screen or feature: `login-screen.png`, `dashboard-empty-state.png`
- Include both desktop and mobile variants where responsive behaviour differs
- Prefer linking to a live Figma/Zeplin project over copying files when designs change frequently
- If a design is finalised and dev-ready, label it clearly: `checkout-flow-FINAL.png`
