# Diagrams

Place architecture and flow diagrams here so Copilot and teammates can reference them during development.

## What belongs here

- **Architecture diagrams** — system overview, component relationships, service boundaries
- **Data flow diagrams** — how data moves through the application
- **Entity-relationship (ER) diagrams** — database schema visuals
- **Sequence diagrams** — interaction flows between components or services
- **State machine diagrams** — UI state transitions or workflow logic

## Recommended formats

- `.png` / `.svg` — rendered exports for quick viewing
- `.drawio` — editable Draw.io source files
- `.mmd` — Mermaid diagram source (version-control friendly, renderable in GitHub)

## Tips

- Name files descriptively: `auth-flow.mmd`, `db-schema-v2.png`
- Keep diagrams up to date when the system changes — stale diagrams are worse than none
- Mermaid (`.mmd`) files are preferred where possible because they diff cleanly in git
